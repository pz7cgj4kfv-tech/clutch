04 — Notifications & GPS
Comment ça marche vraiment ?
Explication technique claire — SOS, position, notifications push
🔔 Notifications — 3 options
🌐

Option A — Web Push

API navigateur native. Gratuit. Fonctionne sur Chrome/Firefox/Android. Ne fonctionne pas sur iOS sans app dans l'App Store (sauf Safari 16.4+ si ajouté à l'écran d'accueil).

GRATUIT Faisable en 1 jour
📱

Option B — Expo Push (app native)

Fonctionne sur iOS ET Android. Notifications natives vraies. Nécessite l'app dans l'App Store. La vraie solution long terme.

GRATUIT App Store 99$/an
✉️

Option C — Email + SMS fallback

Supabase envoie un email "Tu as reçu un Clutch !" Pas idéal mais fonctionne partout. SMS via Twilio : 0.05 CHF/SMS.

Email gratuit SMS payant
💡 Recommandation : Maintenant → implémenter Web Push (gratuit, 1 jour de travail, marche sur Android). Plus tard avec l'app native → Expo Push pour iOS aussi. Les emails Supabase servent de fallback immédiat (déjà configurés).
📍 GPS & Localisation — 3 usages différents

Usage 1 — Disponibilité (quartier)

Le navigateur demande l'accès GPS → coordonnées → Nominatim (OpenStreetMap) convertit en "Lausanne, Ouchy". On stocke uniquement le quartier, pas les coordonnées exactes. Gratuit, sans clé API, respectueux de la vie privée. ✅

Usage 2 — SOS ponctuel (actuel)

GPS → lien Google Maps généré → l'utilisateur choisit de le partager manuellement (copier, WhatsApp, partage natif). Ce n'est pas automatique — c'est l'utilisateur qui décide. Conforme RGPD. ✅

Pas de SMS envoyé automatiquement (coûterait 0.05 CHF/envoi). On garde le contrôle chez l'utilisateur.

Usage 3 — Suivi temps réel (à faire)

Pendant un RDV actif : l'utilisateur active le partage → position envoyée à Supabase toutes les 30 sec → l'autre personne (ou contact de confiance) reçoit un lien qui se rafraîchit automatiquement. Expire quand le check-in est fait ou l'utilisateur arrête.

Implémentation : Supabase Realtime + table `location_shares` + page publique `/share/[token]`. ~2 jours de travail Gratuit

🔐 Important légal GPS : En Suisse (LPD), collecter une position GPS exacte est une donnée personnelle sensible. Il faut : (1) consentement explicite avant activation, (2) expiration automatique, (3) possibilité de couper à tout moment, (4) mentionner dans la politique de confidentialité. Votre design actuel respecte ces règles.
05 — Sécurité
Audit de sécurité complet
Vulnérabilités, risques, et comment les corriger
⚠️ Risque principal : L'app gère des rencontres physiques entre inconnus. Un profil fake, une fausse photo, ou un comportement prédateur peut causer du tort réel. La sécurité ici n'est pas optionnelle.

🔴 Failles actuelles — Critiques

  • Pas de vérification que l'email existe (quelqu'un peut s'inscrire avec n'importe quoi)
  • Pas de vérification téléphone → impossible de bannir quelqu'un vraiment
  • Pas de modération des messages (quelqu'un peut envoyer du contenu inapproprié)
  • Pas de signalement d'utilisateur (bouton "Signaler")
  • Pas de limitation des clutches envoyés (quelqu'un peut spammer)
  • Photos non modérées (quelqu'un peut uploader n'importe quoi)

🟢 Ce qui est bien — Points forts

  • Auth Supabase sécurisée (JWT, bcrypt passwords)
  • RLS (Row Level Security) activé sur les tables
  • Position exacte jamais stockée (seulement quartier)
  • Limite 8 messages par RDV (anti-harcèlement)
  • SOS avec 117/144 intégré
  • Lieux suggérés = lieux publics sûrs uniquement
  • Gratuité pour les femmes = signal de bienveillance
🛡 Fixes sécurité recommandés

1. Rate limiting sur les clutches envoyés

Maximum 3 clutches envoyés par jour. Simple à implémenter : vérifier dans Supabase combien de clutches ont été envoyés par cet utilisateur dans les dernières 24h avant d'autoriser. 2h de travail

2. Signalement utilisateur

Bouton "Signaler" sur chaque profil → envoie une ligne dans la table `reports` → admin voit et peut bannir. Simple et essentiel. 4h de travail

3. Modération photo automatique

À l'upload d'une photo : appel à OpenAI Moderation API (gratuit) ou Google Cloud Vision Safe Search (1000 req/mois gratuit). Si la photo est NSFW → refusée automatiquement. 4h de travail

4. Vérification email obligatoire

Supabase propose déjà la confirmation par email. Il suffit d'activer "Email confirmation required" dans les paramètres Supabase Auth. 10 minutes — juste activer dans les settings Supabase.

5. Score de fiabilité réel

Actuellement c'est une valeur fixe de 100%. Il faudrait le calculer : (clutches acceptés / clutches reçus) × fiabilité passée. Quelqu'un qui fait un "rabbit" (ne se présente pas) voit son score baisser et devient moins visible. 1 jour de travail

06 — Légal
Conformité suisse — Ce qui est obligatoire
LPD (Loi sur la Protection des Données suisse, en vigueur depuis sept. 2023) + RGPD si utilisateurs EU
🚨 Risque légal actuel : Sans CGU ni politique de confidentialité, vous êtes techniquement en infraction avec la LPD suisse (entrée en vigueur sept. 2023). En cas de plainte d'un utilisateur, l'amende peut aller jusqu'à 250 000 CHF. Le risque est faible en bêta avec des amis, mais il faut régler ça avant tout lancement public.

🔴 Obligations immédiates

  • Politique de confidentialité accessible depuis l'app
  • CGU acceptées à l'inscription (case à cocher)
  • Mentions légales (nom de la société ou de la personne responsable)
  • Indiquer que les données sont stockées en Europe (Supabase = Frankfurt)
  • Droit à l'effacement : l'utilisateur doit pouvoir supprimer son compte et toutes ses données

📋 Contenu minimum CGU

  • Qui collecte les données (toi, David Saugy / Clutch)
  • Quelles données (email, prénom, photo, GPS quartier, messages)
  • Pourquoi (fonctionnement de l'app)
  • Où stockées (Supabase, serveurs Frankfurt, EU)
  • Durée de conservation (ex: 2 ans après dernière connexion)
  • Comment exercer ses droits (email de contact)
  • Règles de comportement (pas de contenu illégal, pas de harcèlement)
💡 Solution rapide : Je peux générer un texte CGU et politique de confidentialité adapté à la LPD suisse et pré-rempli avec vos informations. À ajouter sur une page /legal de l'app. Estimé : 2h de travail.
07 — Design & UX
Audit graphique et ergonomique
Ce qui est bien, ce qui peut mieux, et des idées concrètes

✅ Points forts design

  • Identité visuelle cohérente (rose primaire + beige)
  • Frame "téléphone" sur desktop = effet app immédiat
  • Typographie clean, hiérarchie lisible
  • Plein écran sur mobile (100dvh)
  • Animations légères (toggle dispo)
  • Score fiabilité visible (facteur de confiance)

⚠️ Problèmes UX identifiés

  • Liste profils trop sobre — pas assez "vie" dedans
  • Sélecteur "jusqu'à" : dropdown mal ergonomique sur mobile
  • Pas de micro-animations sur les transitions d'écran
  • Onboarding : la photo est optionnelle = beaucoup de profils sans photo
  • Démo : on ne comprend pas immédiatement les scénarios disponibles
  • Pas de splash screen d'attente pendant le chargement des profils
